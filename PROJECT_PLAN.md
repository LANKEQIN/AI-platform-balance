# AI平台余额快捷入口 - 项目计划

## 一、项目概述

### 1.1 项目背景
作为AI爱好者，我们经常使用多个AI平台（火山引擎、阿里百炼、智谱开放平台等）。每次查看余额都需要打开浏览器、输入网址或从众多标签中寻找，操作繁琐。本项目旨在创建一个本地网页作为中转界面，提供一键直达各平台余额页面的功能。

### 1.2 项目目标
- 创建一个本地运行的网页界面
- 集中展示用户拥有的AI平台
- 提供可自定义的快捷按钮，一键直达各平台余额页面
- 支持预填默认链接和用户自定义链接

### 1.3 目标用户
- 使用多个AI平台的开发者/爱好者
- 需要频繁查看各平台余额的用户
- 希望提高效率、减少重复操作的用户

## 二、功能需求

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 平台展示 | 以卡片形式展示各AI平台 | 高 |
| 快捷跳转 | 点击按钮直达对应平台余额页面 | 高 |
| 链接自定义 | 用户可自定义/修改各平台的跳转链接 | 高 |
| 默认链接 | 预设各平台默认余额页面链接 | 高 |
| 本地存储 | 使用localStorage保存用户自定义配置 | 高 |

### 2.2 扩展功能（可选）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 平台分类 | 按类型分类展示（大模型、图像生成等） | 中 |
| 添加平台 | 用户可添加新的平台卡片 | 中 |
| 删除平台 | 用户可删除不需要的平台卡片 | 中 |
| 深色模式 | 支持明暗主题切换 | 低 |
| 搜索过滤 | 快速搜索定位平台 | 低 |

## 四、技术方案

### 4.1 技术选型

| 层面 | 技术 | 理由 |
|------|------|------|
| 前端框架 | 纯HTML/CSS/JavaScript | 轻量、无需构建、直接本地打开 |
| 样式 | CSS + CSS变量 | 简单直观，便于主题定制 |
| 数据存储 | localStorage | 本地持久化，无需后端 |
| 运行方式 | 直接打开HTML文件 | 无需启动服务器，双击即用 |

### 4.2 项目结构

```
API Balance/
├── index.html          # 主页面
├── css/
│   └── style.css       # 样式文件
├── js/
│   ├── config.js       # 平台配置数据
│   ├── storage.js      # 本地存储管理
│   └── app.js          # 主逻辑
├── assets/
│   └── icons/          # 平台图标（可选）
└── PROJECT_PLAN.md     # 项目计划文档
```

## 七、使用说明

### 7.1 使用方式
1. 双击 `index.html` 在浏览器中打开
2. 点击任意平台卡片的"前往余额"按钮
3. 浏览器将在新标签页打开对应平台余额页面
4. 点击"编辑"可自定义该平台的跳转链接

### 7.2 数据存储
- 所有自定义配置保存在浏览器 localStorage 中
- 清除浏览器数据会导致配置丢失
- 可导出配置进行备份（扩展功能）
