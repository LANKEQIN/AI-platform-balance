# AI平台余额快捷入口 - 第二版项目计划

## 一、项目概述

### 1.1 项目背景
第一版已实现基础的余额快捷入口功能，包括平台卡片展示、链接自定义、深色模式等。第二版将在现有基础上，围绕"提升效率"和"增强体验"两个核心方向进行功能扩展。

### 1.2 第二版目标
- **数据可视化**：增加余额历史记录和趋势图表
- **智能提醒**：余额不足时主动提醒
- **使用统计**：分析各平台使用频率
- **快捷操作**：减少操作步骤，提升效率
- **数据安全**：支持配置导入导出和备份

## 三、详细功能设计

### 3.1 余额记录系统

#### 数据结构
```javascript
{
  platformId: string,      // 平台ID
  records: [
    {
      date: string,        // ISO日期格式
      balance: number,     // 余额数值
      note: string         // 备注（可选）
    }
  ]
}
```

#### 功能流程
1. 用户点击"记录余额"按钮
2. 弹出输入框填写当前余额
3. 保存到localStorage
4. 在卡片上显示最新余额和趋势

### 3.2 趋势图表

#### 实现方案
- 使用 Chart.js 或轻量级自研SVG图表
- 在编辑弹窗中增加"历史记录"标签页
- 展示近30天的余额变化曲线

#### 图表类型
- 折线图：余额趋势
- 柱状图：月度消费对比

### 3.3 低余额提醒

#### 提醒方式
- 卡片角标：红色圆点提示
- 数字徽章：显示低余额平台数量
- 列表筛选：一键查看所有低余额平台

#### 配置项
```javascript
{
  warningThreshold: 50,    // 警告阈值（元）
  dangerThreshold: 10,     // 危险阈值（元）
  enableNotification: true // 是否启用提醒
}
```

### 3.4 批量操作

#### 支持的操作
- 批量删除
- 批量修改分类
- 批量打开（在新标签页打开多个平台）

#### 交互设计
- 卡片左上角显示复选框（hover时显示）
- 底部浮动工具栏显示批量操作按钮
- 支持全选/取消全选

### 3.5 数据导入导出

#### 导出内容
- 平台配置
- 余额历史记录
- 用户设置

#### 导入功能
- 支持选择JSON文件
- 导入前预览和冲突检测
- 合并或覆盖选项

### 3.6 快捷键系统

| 快捷键 | 功能 |
|-------|------|
| Ctrl/Cmd + K | 聚焦搜索框 |
| Ctrl/Cmd + N | 添加新平台 |
| Esc | 关闭弹窗 |
| / | 快速搜索 |

## 七、注意事项

1. **兼容性**：Chart.js等库使用CDN引入，确保离线可用性
2. **性能**：余额记录数据量控制，提供清理历史功能
3. **隐私**：所有数据仍本地存储，不上传服务器
4. **迁移**：第一版用户数据无缝升级到第二版
5. **渐进增强**：核心功能不依赖新增库，确保基础功能可用
